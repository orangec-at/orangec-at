# Second Brain Setup Guide

Build a system to capture, organize, and retrieve knowledge.

## PARA Method

### Projects (Active, deadline-driven)
**Examples**:
- Blog redesign (deadline: Feb 1)
- Portfolio update (deadline: Jan 15)
- Job application prep (deadline: ongoing)

**Criteria**: Has a goal and end date

---

### Areas (Ongoing responsibilities)
**Examples**:
- Career development
- Health & fitness
- Frontend skills
- Writing

**Criteria**: No end date, ongoing maintenance

---

### Resources (Topics of interest)
**Examples**:
- Next.js docs
- React patterns
- TypeScript tips
- Design resources

**Criteria**: Reference material

---

### Archives (Completed/inactive)
**Examples**:
- Completed projects
- Old jobs
- Abandoned ideas

**Criteria**: No longer active

---

## Folder Structure

\`\`\`
vault/
├── daily/              # Daily notes, TILs
├── projects/           # Active projects
├── areas/              # Life areas
├── resources/          # Learning material
└── archives/           # Completed
\`\`\`

---

## Linking Strategy

**Bidirectional links**: [[Note Name]]

**Tag system**: #react #nextjs #performance

**MOCs** (Maps of Content):
- Index note linking related concepts
- Example: "Frontend Learning MOC"

---

## Tools

**Obsidian**: Best for linking, local-first
**Notion**: Good for databases, collaboration
**Markdown files**: Simple, portable, version-controlled

---

## Daily Workflow

1. **Morning**: Review yesterday's notes
2. **Throughout day**: Quick captures
3. **Evening**: Process into permanent notes
4. **Weekly**: Review and reorganize
