# Keyword Research Guide

Quick reference for finding high-value keywords using WebSearch.

## WebSearch Query Patterns

### Primary Keyword Discovery
```
"[topic] tutorial"
"how to [action] with [tech]"
"[tech] guide"
"[tech] best practices"
"learn [tech]"
```

### Long-Tail Opportunities
```
"[tech] for beginners"
"[tech] vs [alternative]"
"[tech] common mistakes"
"[tech] [specific use case]"
"why [tech] [benefit]"
```

### Question Keywords
```
"how to [action]"
"what is [concept]"
"when to use [tech]"
"why [problem]"
"where to [action]"
```

## Keyword Evaluation Criteria

**Search Volume**: High enough to drive traffic
**Competition**: Can you outrank existing content?
**Intent Match**: Does it match your content type?
**Relevance**: Related to your expertise?

## Quick Keyword Research Workflow

1. **Start Broad**: WebSearch for main topic
2. **Analyze Top 10**: What keywords appear in titles?
3. **Find Variations**: "people also search for [keyword]"
4. **Go Long-Tail**: Add qualifiers (beginner, advanced, 2025, specific framework)
5. **Check Competition**: Can you provide unique value?

## LSI Keywords (Related Terms)

Find related keywords naturally mentioned in top-ranking content:

Example for "React Performance":
- optimization
- rendering
- memoization
- lazy loading
- code splitting
- bundle size

Include these naturally throughout content.
