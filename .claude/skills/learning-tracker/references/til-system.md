# TIL (Today I Learned) System

Capture daily learning for compounding knowledge growth.

## Why TIL?

**Benefits**:
- Reinforces learning through writing
- Creates searchable knowledge base
- Shows progress over time
- Generates blog post ideas
- Builds public learning record

---

## Daily Habit

**When**: End of day (5-10 minutes)

**Ask**:
- What's one thing I learned today?
- Why does it matter?
- How can I apply it?

---

## Template

\`\`\`markdown
# TIL - YYYY-MM-DD

## [Topic]

**What I Learned**: [One-sentence summary]

**Details**:
- [Key point 1]
- [Key point 2]

**Example**:
\`\`\`[language]
// Code or concrete example
\`\`\`

**Application**: [How you'll use this]
\`\`\`

---

## Categories

**Technical**:
- New language feature
- Library/framework concept
- Performance optimization
- Debugging technique

**Process**:
- Better workflow
- Tool discovery
- Productivity hack

**Career**:
- Communication insight
- Negotiation tactic
- Job search learning

---

## Publishing Options

**Private** (vault/daily/):
- Just for you
- Raw, unpolished

**Public** (GitHub, Dev.to):
- Share with community
- Refined, with context

---

## Weekly Review

Every Sunday:
- Read week's TILs
- Identify patterns
- Extract blog post ideas
- Archive or expand notes
